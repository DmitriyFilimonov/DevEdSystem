Система организации обучения Разработать бэкэнд для системы организации обучения, которая призвана структурировать взаимодействия между преподавателями, студентами и менеджерами. Система должна предоставлять возможность отслеживания посещаемости, выполнения домашних работ и оплаты курса студентами. Кроме того, студенты в системе должны быть объединены в группы и иметь возможность получать фидбэки по выполненным работам и отвечать на них. Подробности: Функциональность по ролям: Все пользователи системы делятся на 6 ролей: Администратор системы, Студент, Менеджер колледжа, Методист, Преподаватель, Тьютор. При этом у одного пользователя может быть несколько ролей. Администратор системы: Добавление/удаление/редактирование пользователей всех ролей Добавление/удаление/редактирование всего Изменение общих настроек системы Менеджер колледжа: Добавление/удаление/редактирование курсов Добавление/удаление/редактирование студентов Добавление/удаление/редактирование групп Отмечание оплат Просмотр всей статистики по всем группам Методист: Добавление/удаление/редактирование курсов (в т.ч. предлагаемых тем) Добавление/удаление/редактирование заданий курса Добавление/удаление/редактирование материалов курса Просмотр всей статистики по всем группам Преподаватель: Просмотр всей статистики по своим группам Добавление/удаление/редактирование занятий Добавление/удаление/редактирование заданий вне курса (может добавлять свои дз, а не только те, которые включены в курс методистом) Добавление/удаление/редактирование материалов вне курса Назначение/добавление/проверка ДЗ для своих групп Тьютор: Просмотр всей статистики по своим группам Проверка ДЗ для своих групп Добавление материалов вне курса Студент: Просмотр своей статистики Просмотр информации о курсе/группе/заданиях/материалах Отправка ответов на ДЗ Оставлять фидбэки к занятиям Дополнительные сущности кроме ролей: Курс: Каждый курс содержит в себе описание, список тем занятий и набор заданий и материалов. Группа: Каждая группа относится к какому-то курсу, содержит преподавателя(лей) и студентов, включает пройденные занятия и назначенные задания и отправленные материалы. Длительность обучения группы не фиксирована. У групп есть статусы. Занятие: Каждое занятие состоит из темы, комментариев преподавателя и списка студентов и отзывов студентов. У одного занятия может быть несколько тем. Задание: Система тэгов для упрощения поиска. Задания подразделяются на 2 вида: требующие решения и не требующие. Задания состоят из дат начала/сдачи, формулировки, статуса (назначено / просрочено / требует проверки / требует исправления / сдано / сдано с опозданием), комментариев и имеют возможность содержать ссылки (явно отдельно, для удобства фронта) Задание не обязательно привязано к конкретному занятию и его теме. Материалы: Текстовые сообщения или ссылки, видимые на странице группы. Система тэгов для упрощения поиска. Материал не обязательно привязано к конкретному занятию и его теме. Платежи: Кол-во платежей считается по запланированному числу недель на курсе (n/4 с округлением вниз)
